LE PROJET
LE CONTEXTE :
L'entreprise Symfeed désire se lancer dans un nouveau marché, elle hésite entre plusieurs secteurs et décide de lancer
un appel d'offres auprès de plusieurs Universités Françaises dans le but de réaliser plusieurs POC (Proof Of Concept ->
démonstration de faisabilité).

L'Université Paris 1 Panthéon-Sorbonne a remporté cet appel d'offres et vous missionne dans le cadre de votre formation
Symfony 5 afin d'étudier et de réaliser ce projet.

LES NIVEAUX :
I- Bronze (plafond de la note 10/20) :


Utiliser un outil de versionning (type Git) avec l'utilisation de branches correspondant à des tickets précis dans votre backlog.
Aucun commit ne doit être effectué directement sur la branche master ;
Travailler en méthodologie agile (choisir un outil (type Trello) pour la gestion de vos sprints) ;
Définir un Product Owner (PO) 1 ;
Avoir un minimum de 5 fonctionnalités (que vous devez faire valider à votre client avant de débuter la phase de développement) ;
Utiliser un Framework CSS (type Bootstrap, Materialize, Foundation, etc.) ;
Utiliser une librairie tierce (disponible via le site Packagist) ;
Utiliser le gestionnaire de paquets "Composer" ;
Avoir un minimum de 3 tables dans votre base de données ;
Utiliser une base de données MySQL ou NoSQL (au choix) ;
Utiliser le moteur de templates Twig (héritage, tags, filtres, etc.) ;
Utiliser l'ORM Doctrine : les entités et les repositories ;
Rédiger une documentation technique (présentation des langages/technologies utilisées, présentation de l'architecture, MCD, etc.) ;
Utiliser des contrôleurs ;
Utiliser des routes ;
Utiliser des fichiers de configuration et paramètres ;
Code HTML passant la validation W3C (aucune erreur ne doit être présente) ;
Code CSS passant la validation W3C CSS (aucune erreur ne doit être présente).

II- Gold (plafond de la note 15/20) :


Utiliser des classes services avec Symfony ;
Utiliser l'injection de dépendances ;
Utiliser les formulaires ;
Utiliser le composant Validator de Symfony ;
Utiliser les fixtures ;
Création de requêtes personnalisées avec l'ORM Doctrine ;
Système d'envoi de mail (avec le composant mailer de Symfony).

III- Challenger (plafond de la note 20/20) :

Utilisation d'une API tierce ;
Présence de tests unitaires ;
Présence de tests fonctionnels ;
Ajout de l'internationalisation (i18n) ;
Création de comptes utilisateurs et authentification ;
Sécuriser l'application avec un système de rôles ;
Mise en place d'un back-office pour l'administration ;
Utilisation du composant Event Listener ou Event Dispatcher de Symfony.
